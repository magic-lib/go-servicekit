## 2026-06-08 22:15

创建了 `generate_assets.go` 工具，用于自动生成 `assets.go` 文件。

功能：
- 使用 `embed` 将模板文件 `assets.tmpl` 嵌入到可执行文件中，无需外部模板。
- 输入目录名，扫描该目录下的所有文件（递归，跳过 `assets.go` 本身）。
- 生成 `assets.go` 文件，包含每个文件的 `//go:embed` 指令、常量定义和映射表。
- 常量名根据文件名自动转换为大写驼峰格式（去除扩展名，替换下划线和连字符，单词首字母大写）。
- 支持可选的包名参数（第二个参数），默认使用目录名，无效时回退到 "static"。
- 生成的 `assets.go` 文件包含 `LoadFiles` 函数，用于在运行时读取嵌入的文件。

使用方法：
```bash
# 生成 assets.go 到指定目录
go run generate_assets.go <目录> [包名]

# 编译成独立二进制
go build -o genassets generate_assets.go
./genassets <目录> [包名]
```

该工具已测试，可以独立运行，不依赖外部模板文件。

## 2026-06-08 23:17

已扩展 `generate_assets.go` 功能以支持多目录和模板路径：

1. **多目录支持**：`-dir` 参数现在支持逗号分隔的多个目录（例如 `-dir dir1,dir2,dir3`）
2. **模板路径支持**：`-tpl_path` 参数支持逗号分隔的模板路径前缀，与目录一一对应
   - 如果提供单个值，应用于所有目录
   - 如果提供多个值，必须与目录数量相同
   - 如果留空，则不添加前缀
3. **智能常量命名**：`filenameToConstWithPrefix` 函数根据模板路径前缀生成唯一的常量名，避免不同目录下同名文件的冲突
4. **输出目录控制**：`-go_out` 参数指定生成的 `assets.go` 文件输出目录
5. **包名逻辑**：如果没有指定包名，使用 `-go_out` 的基名或第一个输入目录的基名

新功能示例：
```bash
# 多目录，对应模板路径
./genassets_multi -dir templates/common,templates/special -tpl_path common,special -go_out generated

# 单目录，带模板前缀
./genassets_multi -dir templates -tpl_path mytemplates -pkg mypkg

# 向后兼容的位置参数
./genassets_multi templates mypackage
```

核心改进：
- 使用 `flag` 包进行健壮的命令行解析
- 递归扫描多个目录，收集所有文件
- 为每个文件生成包含模板路径的嵌入路径
- 常量名包含前缀以避免命名冲突
