# 2026-06-11

## templatefile/generate_assets.go - 多项修复

### 1. EmbedPath 计算方式修正
- `EmbedPath` 应该是相对于 `goOut`（生成的 assets.go 所在目录）的相对路径，而不是相对于控制台工作目录。
- 改为使用 `filepath.Rel(outputDir, fullPath)` 来计算，直接算出从 `outputDir` 到源文件的实际相对路径。

### 2. outputDir 转绝对路径
- `goOut` 可能是相对路径，而 `fullPath` 是绝对路径，`filepath.Rel` 要求两者同类型。
- 在确定 `outputDir` 后对其做 `filepath.Abs` 转换。

### 3. ConstName 根据 EmbedPath 生成
- 之前用 `filenameToConstWithPrefix(cf.RelPath, cf.TplPrefix)` 生成常量名，不同目录下同名文件可能冲突。
- 改为用 `embedPathToConst(embedPath)`，将完整的 EmbedPath（含目录层级）转成 CamelCase 常量名。
- 删除了旧的 `filenameToConstWithPrefix` 函数，新增 `embedPathToConst` 函数。

### 4. Dir 字段从 EmbedPath 提取
- `Dir` 字段通过 `filepath.Dir(embedPath)` 从 EmbedPath 中提取目录部分，根目录文件则为空字符串。

### 5. 新增 -go_file 参数
- 新增 `-go_file` flag，允许自定义生成的 Go 文件名，默认为 `assets.go`。
- 跳过逻辑同步更新，不再硬编码跳过 `assets.go`，改为跳过指定的输出文件名。

### 6. allFiles 增加目录通配符条目
- 在 `allFiles` 最前面插入目录通配符条目（如 `js/*`），对应 `//go:embed js/*`。
- 通配符条目排在所有单个文件之前，使得生成的代码中可以先 `fs.Sub(embedFS, "js")` 来获取整个子目录的文件系统。
- `collectedFile` 新增 `IsDirWild` 字段区分通配符条目和普通文件条目。
- 通配符条目不读取实际文件信息（`os.Stat`），而是用目录名作为标识。

### 7. 新增 AssetDir 和 LoadAssetDir
- `fileInfo` 新增 `IsDirWild` 字段传给模板。
- 模板中将通配符目录条目单独存入 `allDirs` map（`map[string]*AssetDir`），普通文件仍在 `allFiles`。
- 新增 `AssetDir` 类型（Key、DirName、EmbedFS），新增 `LoadAssetDir(dirName)` 函数，内部使用 `fs.Sub` 提取子目录文件系统。
- 新增 `dirCacheData` 缓存目录文件系统，避免重复 `fs.Sub`。
